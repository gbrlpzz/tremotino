# Tremotino Research Moat Direction

Tremotino is not meant to become another generic notes app. It is the local operating layer for a personal research moat:

- durable project memory instead of scattered chat history
- source-backed claims instead of loose summaries
- reusable arguments for grants, outreach, reports, and applications
- people/institution context for research-fit conversations
- workflow records for rebuilds, exports, scans, and decisions
- reviewable agent updates instead of automatic memory pollution

## V1 Shape

The first version keeps this small:

- Markdown remains the source of truth.
- The app gives a fast inbox and project/review surfaces.
- Agents can search/fetch context and propose updates through MCP.
- Local project scans create review proposals.
- The MCP registry monitor keeps the system aware of protocol movement.

## Later Shape

Future versions should add typed memory views for:

- sources
- claims
- projects
- people
- institutions
- applications
- datasets
- workflows
- decisions

Each typed object should still compile down to plain Markdown so the vault stays portable.
